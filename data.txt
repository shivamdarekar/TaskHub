========================================
BACKEND SETUP
========================================
- create folders frontend & backend
- cd backend -> npm init

Production Dependencies:
- npm i express dotenv cors
- npm i socket.io @prisma/client
- npm i bcryptjs jsonwebtoken cookie-parser
- npm i helmet express-rate-limit (security)
- npm i nodemailer axios
- npm i razorpay
- npm i redis
- npm i zod

Dev Dependencies:
- npm install -D typescript ts-node-dev prisma
- npm install -D @types/node @types/express @types/jsonwebtoken
- npm install -D @types/bcrypt @types/cookie-parser @types/nodemailer

Setup & Configuration:
- setup scripts in package.json:
  "dev": "ts-node-dev --respawn --transpile-only src/app.ts"
  "build": "tsc"
  "start": "node dist/app.js"
  "prisma:generate": "prisma generate"
  "prisma:migrate": "prisma migrate dev"
  "prisma:deploy": "prisma migrate deploy"

- npx tsc --init -> setup tsconfig.json
- npx prisma init
- setup database connection in .env (DATABASE_URL)
- create prisma schema models
- npx prisma migrate dev --name init
- npx prisma generate

Note: Why not setup prisma inside src?
- Prisma migration always looks for prisma/schema.prisma
- Keeping schema.prisma inside src/ makes migrations/config messy

Backend Scripts:
- npm run dev: Run development server with hot reload
- npm run build: Compile TypeScript to JavaScript
- npm start: Run production server
- npm run prisma:generate: Generate Prisma Client
- npm run prisma:migrate: Create and apply new migrations
- npm run prisma:deploy: Apply migrations in production

========================================
DOCKER CONFIGURATION & DEPLOYMENT
========================================
Backend Dockerization:
1. Create Dockerfile and .dockerignore in backend folder
2. docker build -t taskhub-backend .
   -> Builds Docker image from Dockerfile
3. docker run -p 5000:5000 --env-file .env taskhub-backend
   -> Test container locally with environment variables
4. Create account on Docker Hub (hub.docker.com)
5. docker login
   -> Enter username and password
6. docker tag taskhub-backend shivamdarekar/taskhub-backend
   -> Docker Hub requires images to follow username/repository-name format
   -> Creates alias, both names point to same image
7. docker push shivamdarekar/taskhub-backend
   -> Push image to Docker Hub
8. Deploy from Railway/Render using image URL: shivamdarekar/taskhub-backend

Important Notes:
- Docker images are immutable; rebuild after code changes
- Use --env-file or -e flags to pass environment variables
- Remove quotes from DATABASE_URL in .env for Docker compatibility
- Set all required env vars in Railway: DATABASE_URL, REDIS_*, RAZORPAY_*, etc.


========================================
CI/CD WITH GITHUB ACTIONS
========================================
Automate Docker image builds and pushes using GitHub Actions.

Setup GitHub Actions Workflow:
1. Create workflow file:
   - Location: .github/workflows/backend-docker.yml (must be in repo root)

2. Get Docker Hub credentials:
   - get access token from docker hub account settings

3. Add GitHub Repository Secrets:
   - Go to your GitHub repo
   - Navigate to: Settings → Secrets and variables → Actions
   - Click "New repository secret"
   - Add DOCKER_USERNAME: Your Docker Hub username
   - Add DOCKER_TOKEN: The access token from step 2

How It Works:
- Triggers automatically when backend/** files are pushed
- Runs on GitHub's free Ubuntu servers (Docker pre-installed)
- Builds Docker image using your Dockerfile
- Pushes to Docker Hub with multiple tags
- No need to run Docker Desktop locally!

Tags Created:
- Branch push: main, main-abc123 (commit SHA), latest
- Version tag (v1.0.0): 1.0.0, 1.0, 1, latest

Benefits:
✓ Zero manual work after setup
✓ Automatic builds on every push to backend
✓ Version control for all images
✓ 100% FREE (GitHub Actions + Docker Hub free tiers)
✓ Docker layer caching for 68% faster builds
✓ No Docker Desktop needed on local machine

Cost:
- GitHub Actions: FREE for public repos, 2000 min/month for private
- Docker Hub: FREE for unlimited public repositories
- Typical usage: ~150 minutes/month (within free tier)
